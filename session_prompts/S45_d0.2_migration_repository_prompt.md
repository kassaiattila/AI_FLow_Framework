```

  intake_packages:     package_id (PK), source_type, tenant_id, status, source_metadata (JSONB),

                       role, association_confidence, association_method

Indexek:
  idx_intake_packages_tenant, idx_intake_packages_status, idx_intake_packages_created
  idx_intake_files_package, idx_intake_files_sha256
  idx_intake_descriptions_package

FONTOS:
  - /aiflow-database skill szabályok: zero-downtime, nullable=True new columns
  - Alembic revision generálás a meglévő migrations/ struktúrában
  - Teszteld: upgrade → data insert → downgrade → re-upgrade ciklus
  - down_revision: nézd meg a jelenlegi head-et `alembic current`-tel!

### LÉPÉS 3: IntakeRepository (Day 4-5, 106_ Section 4.4)

```
Cél: asyncpg-based CRUD repository az intake modulhoz.

Fájl: src/aiflow/state/repositories/intake.py

Metódusok:
  - insert_package(package: IntakePackage) → None  (atomic: package + files + descriptions)
  - get_package(package_id: UUID) → IntakePackage | None  (full hydration)
  - transition_status(package_id, new_status) → None  (SELECT FOR UPDATE + validate)
  - list_packages(tenant_id, status?, limit, offset) → list[IntakePackage]

Forrás: 106_ Section 4.4.1

FONTOS:
  - asyncpg pool injection (nem SQLAlchemy ORM — raw SQL)
  - Idempotens transition (state machine validation)

  - Tenant isolation: minden query-ban WHERE tenant_id = $N
