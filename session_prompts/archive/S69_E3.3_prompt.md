## FELADATOK

### LEPES 1: Multi-source adapter matrix E2E

```
Cel:    Mind az 5 adapter (Email, File, Folder, Batch, Api) ugyanazt
        az IntakePackage domain-t termeli + PolicyEngine profile_a alatt route-olodik
Fajl:   tests/e2e/v1_4_1_phase_1b/test_multi_source_e2e.py (un-skip + implementacio)

Kotelezo scenariok (pytest.parametrize `source_type`):
  - [ ] email:    EmailSourceAdapter.enqueue(.eml fixture) -> IntakePackage(source_type=email)
  - [ ] file:     FileSourceAdapter.enqueue(single file)   -> IntakePackage(source_type=file_upload)
  - [ ] folder:   FolderSourceAdapter.enqueue(watched dir) -> IntakePackage(source_type=folder_import)
  - [ ] batch:    BatchSourceAdapter.enqueue(zip archive)  -> IntakePackage(source_type=batch_import)
  - [ ] api:      ApiSourceAdapter.enqueue(hmac-signed bytes) -> IntakePackage(source_type=api_push)

Validacio adapter-enkent:
  - package.source_type egyezik az adapter IntakeSourceType ClassVar-javal
  - len(package.files) >= 1
  - PolicyEngine.route(package).routing_decision nem None
  - IntakeRepository.insert_package() + get_package() round-trip intact
  - a hydratalt package field-ek megegyeznek a pre-insert package-gel (mime, sha256, size)

Minta fixture reuse: tests/e2e/v1_4_1_phase_1b/fixtures/ — mar letezik Week 1+2-bol
Tenant isolation: egy parametrized tenant_id-t hasznalj source_type-onkent + fixture teardown
```
