## FELADATOK

### LEPES 1: `BatchSourceAdapter` implementacio

```
Cel:    ZIP/tar → unpack → per-file IntakePackage emission
Fajl:   src/aiflow/sources/batch_adapter.py (uj)
Forras: 01_PLAN/session_S55_*.md Day 7 (E2.2),
        101_AIFLOW_v2_COMPONENT_TRANSFORMATION_PLAN.md R1

API:
  BatchSourceAdapter(
      storage_root: Path,
      tenant_id: str,
      max_archive_bytes: int = 500_000_000,  # 500 MB
      max_file_count: int = 100,
      max_compression_ratio: float = 50.0,   # ratio guard
      max_nested_depth: int = 3,             # nested archive depth
      glob_patterns: list[str] | None = None,
      excluded_patterns: list[str] | None = None,
  )

  metadata:       SourceAdapterMetadata(source_type=BATCH_IMPORT, transport="push",
                                        requires_ack=False, supports_batching=True)
  enqueue(*, archive_path: Path | None, raw_bytes: bytes | None, filename: str)
      → list[IntakePackage]   # N fajlhoz N package
  async fetch_next() -> IntakePackage | None
  async acknowledge(package_id)
  async reject(package_id, reason)
  async health_check() -> bool

Kulcs invariansok:
  - push transport (like FileSourceAdapter): external caller provides archive via enqueue()
  - enqueue() returns ALL extracted packages at once (unlike folder: 1 file → 1 package lazily)
  - ZIP: zipfile.ZipFile, TAR: tarfile.open (mode='r:*' for auto-detect compression)
  - Zip-bomb guard: (uncompressed_total / compressed_archive_size) > max_compression_ratio → reject
  - Symlink/path traversal: skip entries with '..' or absolute paths, log warning
  - Directories inside archive: skip silently
  - Empty archive (no files after filtering): SourceAdapterError
  - Corrupt archive: SourceAdapterError with descriptive message
  - Per-file: copy to storage_root/{tenant}/{package_id}/{sanitized_filename}
  - Each file gets its own UUID package_id, sha256, size_bytes

TILOS:
  - shutil.unpack_archive (no security controls)
  - extractall() without path validation
  - Reading entire archive into memory if >max_archive_bytes
```
