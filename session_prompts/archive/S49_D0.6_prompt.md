## FELADATOK

### LEPES 1: InstanceConfig.policy_override mezo

```
Cel: InstanceConfig bovitese policy_override mezovel a per-instance policy config-hoz.

Fajl: src/aiflow/skill_system/instance.py

Bovites:
  InstanceConfig-hoz uj mezo:
    policy_override: dict[str, Any] | None = None

FONTOS:
  - Meglevo mezok NE valtoznak (backward compat)
  - A mezo opcionalis (None default) — meglevo YAML-ek tovabbra is mukodnek
  - A dict tartalma PolicyConfig field override-ok (pl. {"cloud_ai_allowed": true})
```

### LEPES 2: PolicyEngine.get_for_instance()

```
Cel: PolicyEngine bovitese instance-level override merge-dzsel.

Fajl: src/aiflow/policy/engine.py (bovites)

Uj metodus:
  def get_for_instance(self, tenant_id: str, instance_override: dict[str, Any] | None = None) -> PolicyConfig:
      """Return merged config: profile defaults + tenant override + instance override."""

Merge lancol:
  1. Profile base config (from YAML)
  2. Tenant-level override (from DB / dict)
  3. Instance-level override (from InstanceConfig.policy_override)

FONTOS:
  - get_for_tenant() NE valtozzon
  - get_for_instance() az uj entry point az instance-szintu policy-hoz
  - Ha instance_override None → get_for_tenant() eredmenyet adja vissza
```

### LEPES 3: instance_loader.py bovites

```
Cel: Instance loader bovitese hogy a policy_override-ot is betoltse.

Fajl: src/aiflow/skill_system/instance_loader.py

Bovites:
  - YAML betoltesnel a policy_override mezo is legyen kezelve
  - Ha a YAML-ben van policy_override szekció → dict-kent toltse be
```

### LEPES 4: Unit tesztek (10+)

```
Fajl: tests/unit/skill_system/test_instance_policy.py (uj)

Minimum tesztek:
  - InstanceConfig.policy_override defaults to None
  - InstanceConfig with policy_override dict
  - PolicyEngine.get_for_instance() without instance override → tenant merge
  - PolicyEngine.get_for_instance() with instance override → triple merge
  - PolicyEngine.get_for_instance() instance override trumps tenant override

  - Backward compat: existing InstanceConfig YAML without policy_override still works
```
