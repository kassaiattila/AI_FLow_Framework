## FELADATOK (default: Graph API backend)

### LEPES 1: GraphApiBackend core

```
Cel:    Microsoft Graph /me/mailFolders/inbox/messages polling —
        GraphApiBackend implements ImapBackendProtocol kontraktot.
Forras: 01_PLAN/session_S55_*.md Day 3 (E1.2-A)
Fajlok:
  - src/aiflow/sources/graph_backend.py (uj)
      * GraphApiBackend:
        - auth: msal ConfidentialClientApplication (client_credentials flow)
        - fetch_unseen() → GET /users/{user}/mailFolders/inbox/messages?$filter=isRead eq false
            - Download raw MIME via /messages/{id}/$value endpoint
            - Return [(message_id_as_int_hash, raw_bytes)]
              (UID = stable int hash of Graph message id; in-flight map tolerates strings too)
        - mark_seen(uid: int) → PATCH /messages/{id} {"isRead": true}
        - mark_flagged(uid, reason) → PATCH /messages/{id} {"flag": {"flagStatus": "flagged"}}
          + categories.add(reason) for audit trail
        - ping() → GET /me (or service principal ping)
      * Dependency: msal>=1.26, httpx>=0.27 (already in uv.lock — verify)

Architektura: GraphApiBackend az `ImapBackendProtocol`-t implementalja, igy az
EmailSourceAdapter valtozatlan marad — csak a backend injektalas different.

Tenant-keyed: a backend config tartalmazza: tenant_id (MS tenant), client_id,
client_secret (Vault-bol!), user_principal_name vagy mailbox UPN.
```
