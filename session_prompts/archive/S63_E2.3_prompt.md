### LEPES 1: `ApiSourceAdapter` implementacio

```
Cel:    Webhook push adapter — HMAC-SHA256 verify, replay window, idempotency
Fajl:   src/aiflow/sources/api_adapter.py (uj)
Forras: 01_PLAN/session_S55_*.md Day 8 (E2.3-A),
        101_AIFLOW_v2_COMPONENT_TRANSFORMATION_PLAN.md R1

API:
  ApiSourceAdapter(
      storage_root: Path,
      tenant_id: str,
      hmac_secret: str,                           # shared secret
      signature_header: str = "X-Webhook-Signature",
      timestamp_header: str = "X-Webhook-Timestamp",
      max_clock_skew_seconds: int = 300,           # 5 min
      max_package_bytes: int | None = None,
      adapter_name: str = "api_push",
      adapter_version: str = "0.1.0",
  )

  metadata:       SourceAdapterMetadata(source_type=API_PUSH, transport="push",
                                        requires_ack=False)
  enqueue(*, payload: bytes, filename: str, signature: str, timestamp: str,
          idempotency_key: str | None = None, mime_type: str | None = None)
      → IntakePackage
  async fetch_next() -> IntakePackage | None
  async acknowledge(package_id)
  async reject(package_id, reason)
  async health_check() -> bool

Kulcs invariansok:
  - HMAC-SHA256: hmac.compare_digest(expected, received) — timing-safe
  - Signature payload: timestamp + "." + base64(body) — prevents replay
  - Replay window: abs(now - timestamp) > max_clock_skew_seconds → reject
  - Idempotency: ha key mar latott → SourceAdapterError (double-delivery guard)
  - push transport (like FileSourceAdapter): single file per enqueue
  - Storage: storage_root/{tenant}/{package_id}/{sanitized_filename}

TILOS:
  - Plain string comparison for HMAC (timing attack)
  - Storing hmac_secret in logs
  - Accepting webhooks without signature verification
```
