## FELADATOK

### LEPES 1: IntakePackage.association_mode attribute

```
Cel:    IntakePackage Pydantic mezo + DB rogzites
Fajl:   src/aiflow/intake/package.py (modositas)

Valtozas:
  - association_mode: AssociationMode | None = Field(None, description="How files were associated to descriptions (N4).")
  - Import csak a TYPE_CHECKING-ben VAGY forward ref string ("AssociationMode"), hogy ne legyen kororos import (associator -> package).
  - Alternativ: Literal["explicit", "filename_match", "order", "single_description"] | None — es runtime-ban convert.
```

### LEPES 2: BatchSourceAdapter associator integration

```
Cel:    BatchSourceAdapter hasznalja az N4 associatort amikor descriptions + files egyszerre kerulnek be
Fajl:   src/aiflow/sources/batch_adapter.py (modositas)

API (javasolt):
  async def enqueue(
      self,
      archive_path: Path,
      *,
      tenant_id: str,
      descriptions: list[IntakeDescription] | None = None,
      association_mode: AssociationMode | None = None,
      filename_rules: list[tuple[str, UUID]] | None = None,
      explicit_map: dict[UUID, UUID] | None = None,
  ) -> IntakePackage:
      ...
      mapping = associate(pkg, mode=association_mode, ...)  # ures package -> ures dict
      for desc in pkg.descriptions:
          desc.associated_file_ids = [fid for fid, did in mapping.items() if did == desc.description_id]
      pkg.association_mode = association_mode
      return pkg

Ha hiba → AssociationError visszaadasa (ne swallow-oljuk, az E3.1-B kontextusban a caller feldolgozhatja).
```
