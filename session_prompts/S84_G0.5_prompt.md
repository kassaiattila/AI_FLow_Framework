## FELADATOK

### LEPES 1 — Webhook router (`POST /api/v1/sources/webhook`)

Add `src/aiflow/api/v1/sources_webhook.py`:
- Endpoint: `POST /api/v1/sources/webhook` (raw bytes, multipart NOT used; webhook senders post raw body + signature/timestamp headers).
- Headers: `X-Webhook-Signature`, `X-Webhook-Timestamp`, optional `X-Idempotency-Key`.
- HMAC secret from env: `AIFLOW_WEBHOOK_HMAC_SECRET` (already used by `_warmed_app` in test_upload_package E2E).
- `tenant_id` resolution: same pattern as `intake.py:get_tenant_id` (JWT `team_id` claim).
- Route handler:
  1. Build `ApiSourceAdapter` per request (or singleton with shared in-memory idempotency store — decide in-session).
  2. `adapter.enqueue(payload=body, filename=..., signature=..., timestamp=..., idempotency_key=...)` → `IntakePackage`.
  3. `await IntakePackageSink(repo=repo).handle(package)` — single persistence path.
  4. Return 201 with `{package_id, source_type, file_count, status}`.
- Error mapping:
  - `SourceAdapterError("missing signature")` → 400
  - `SourceAdapterError("invalid HMAC signature")` → 401
  - `SourceAdapterError("timestamp outside replay window")` → 401
  - `SourceAdapterError("duplicate idempotency_key")` → 409

Register the router in `src/aiflow/api/app.py` next to the intake router.
