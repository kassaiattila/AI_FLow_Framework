## FELADATOK

### LEPES 1: FileSourceAdapter core

```
Cel:    Single-file source adapter — input = (file_path | bytes + filename),
        output = IntakePackage with exactly 1 IntakeFile (no descriptions unless
        provided). No multipart, no association — that lives in N4 (E3.1).
Forras: 01_PLAN/session_S55_*.md Day 4 (E1.2 in plan doc), 101_*.md R1.
Fajlok:
  - src/aiflow/sources/file_adapter.py (uj)
      * FileSourceAdapter(SourceAdapter):
        - source_type = IntakeSourceType.FILE_UPLOAD
        - metadata: transport="push", requires_ack=False, supports_batching=False
        - __init__(*, storage_root, tenant_id, max_package_bytes=None,
                    mime_detect: Callable[[bytes, str], str] | None = None,
                    adapter_name="file_upload", adapter_version="0.1.0")
        - enqueue(*, file_path: Path | None = None,
                  raw_bytes: bytes | None = None,
                  filename: str,
                  mime_type: str | None = None,
                  description: str | None = None) -> IntakePackage
            # Push-mode helper: bridges an external uploader into the internal
            # queue. Returns the IntakePackage synchronously so callers
            # (REST endpoint, CLI, tests) have the package_id immediately.
        - fetch_next() -> IntakePackage | None
            # Drains an internal deque<IntakePackage> filled by enqueue().
        - acknowledge(package_id) -> None (no-op but clears in-flight map)
        - reject(package_id, reason) -> logs and discards; no upstream to flag.
        - health_check() -> True if storage_root writable.
      * MIME detection: default via stdlib `mimetypes.guess_type(filename)`;
        fall back to "application/octet-stream". Optional `mime_detect` DI for
        libmagic upgrade path.
      * Size guard: enforces max_package_bytes; oversize → reject + raise
        SourceAdapterError (push semantics — caller needs immediate feedback).
      * Storage: bytes spilled to {storage_root}/{tenant_id}/{package_id}/{filename}
        with same sanitize_filename helper used by email_adapter (extract to
        sources/_fs.py helper to share).

Architektura: FileSourceAdapter is "push" transport — unlike IMAP pull, the
external uploader creates the package synchronously. fetch_next() + ack/reject
still exist for a uniform Registry/contract-test surface, but most callers
use enqueue() directly. requires_ack=False reflects that there is no upstream
to confirm back to.
```
