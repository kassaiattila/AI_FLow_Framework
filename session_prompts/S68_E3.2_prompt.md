### Hova tartunk — Week 3 Day 13 (E3.2)
A canonical `POST /api/v1/intake/upload-package` multipart endpoint, amin keresztul kulso uploaderek kerik a multi-file + multi-description package-et. Ez az egyetlen "push-mode" belepesi pont amihez nem kell sem email sem batch archive — direkt multipart POST.

1. `src/aiflow/api/v1/intake.py`-ban (ha letezik, egyebkent uj modul) egy `POST /upload-package` endpoint
2. Multipart form fields:
   - N `files[]` field (egyenkent IntakeFile-la valik)
   - M `descriptions[]` JSON field (egyenkent IntakeDescription)
   - `association_mode`: optional (`explicit` | `filename_match` | `order` | `single_description`)
   - `filename_rules`: optional JSON array `[{"pattern": ..., "description_id": uuid}]`
   - `explicit_map`: optional JSON `{file_id: description_id}` (FE-bol valoszinutlen; hagyjuk opcionalisnak)
   - `tenant_id` JWT claim-bol (NEM form field!)
3. Handler epit IntakePackage-et (source_type=file_upload), meghivja associate()-et, IntakeRepository.insert_package()-et, 201 Created + package summary JSON (files minus raw paths).
4. `tests/integration/intake/test_upload_package_router.py` (uj) — valos FastAPI TestClient + valos Postgres.
5. `tests/e2e/v1_4_1_phase_1b/test_upload_package.py` — **un-skip** a ket meglevo placeholder test-et.
6. `docs/api/openapi.yaml` (ha letezik) — upload-package endpoint leiras + pelda request.

## FELADATOK

### LEPES 1: Upload-package router + multipart handler

```
Cel:    POST /api/v1/intake/upload-package - multipart ingestion -> IntakePackage
Fajl:   src/aiflow/api/v1/intake.py (uj VAGY extend ha letezik)
        src/aiflow/api/v1/__init__.py (router registracio)

Signature (javasolt FastAPI):
  @router.post("/upload-package", status_code=201, response_model=UploadPackageResponse)
  async def upload_package(
      files: list[UploadFile],
      descriptions: str = Form("[]"),       # JSON-encoded list[IntakeDescription-like dict]
      association_mode: str | None = Form(None),
      filename_rules: str | None = Form(None),    # JSON array
      explicit_map: str | None = Form(None),      # JSON dict
      request: Request = ...,
      tenant_id: str = Depends(tenant_from_jwt),  # meglevo dep
      repo: IntakeRepository = Depends(get_intake_repo),
  ) -> UploadPackageResponse:
      ...

Logika:
  1) files + descriptions -> IntakeFile/IntakeDescription list (mint E3.1-B BatchAdapter _build_merged_package, de direkt multipart form-bol)
  2) associate() hivasa (EmptyFallbackMode ha nincsenek descriptions)
  3) repo.insert_package()
  4) UploadPackageResponse: package_id, association_mode, file_count, description_count, associations (description_id -> file_id list)

Storage: /var/lib/aiflow/uploads/{tenant_id}/{package_id}/ (configurable)
Hibakeles: 400 rossz JSON / rossz mode, 413 ha >max_total_bytes, 401 ha tenant_id hianyzik, 422 ha associate() AssociationError-t dob
```
