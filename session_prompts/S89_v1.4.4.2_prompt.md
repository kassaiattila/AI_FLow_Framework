### LEPES 4 — Untitled UI decision ADR

Context (from audit, 2026-04-25):
- Legacy `aiflow-admin/src/components/` uses Untitled UI icons (`@untitleduI/icons`) + wraps older layout primitives.
- New `aiflow-admin/src/pages-new/` is pure Tailwind v4, 0% Untitled UI components.
- No ADR exists explaining the divergence.
- Every future UI session will ask "which path?" without this decision on record.

Options:
1. **Adopt** — rebuild `pages-new/` on Untitled UI components. Cost: ~2-3 UI sessions. Benefit: single design system, polished defaults.
2. **Reject** — commit to Tailwind v4 + custom components (`components-new/`), retire Untitled UI. Cost: license audit, icon migration. Benefit: zero external dep coupling, full control.
3. **Hybrid** — keep Untitled UI icons (cheap), build layouts in Tailwind v4 (current `pages-new/` direction). Cost: none (already the current de facto state). Benefit: preserves sunk cost, low friction.

Write `01_PLAN/ADR-UI-Library.md` with:
- Context (as above)
- Decision (**recommend Option 3 / Hybrid** unless you see a strong reason otherwise — it matches the observed current state and minimizes churn)
- Consequences (what future UI sessions must follow)
- Rejected alternatives

Keep it under ~200 lines. ADRs are cheap; the point is to lock the choice on paper.
