## FELADATOK

### LÉPÉS 1 — `AzureDocumentIntelligenceParser` (ParserProvider)

Fájl: `src/aiflow/providers/parsers/azure_document_intelligence.py` (új)

- Implement `ParserProvider` ABC, `PROVIDER_NAME = "azure_document_intelligence"`.
- Dep: `azure-ai-documentintelligence>=1.0` — **opcionális extras-ba**: `pyproject.toml [project.optional-dependencies] azure-di = ["azure-ai-documentintelligence>=1.0"]`. `uv add --optional azure-di azure-ai-documentintelligence`.
- Wrapper: `DocumentIntelligenceClient` + `begin_analyze_document("prebuilt-layout", ...)` → map to `ParserResult`.
- Config (`AzureDIConfig` Pydantic): `endpoint: str | None`, `api_key: str | None` (secret), `default_model_id: str = "prebuilt-layout"`, `timeout_s: int = 120`. Default-et env-ből húzza (`AZURE_DOC_INTEL_ENDPOINT`, `AZURE_DOC_INTEL_KEY`).
- `supports_mime()`: `application/pdf`, `image/png`, `image/jpeg`, `image/tiff`. (A többit Docling jobban csinálja helyben.)
- `estimate_cost(file)`: 0.001 USD/oldal becslés (page_count ismeretlen → `size_bytes / 100_000` floor).
- `async parse()`: `asyncio.to_thread` + SDK sync client; tables → `ParserResult.tables` `{index, markdown, caption, page_number}` alakban.
- Regisztráció: bővítsd `register_default_parsers(registry)` helpert — azureDI **feltételesen** regisztrálva (try/except ImportError), hogy dep hiányában a gate ne törjön.

### LÉPÉS 2 — Router: scan-aware signal + Azure DI routing rule

Fájl: `src/aiflow/routing/router.py` (módosítás)

Új signals `RoutingDecision.signals`-be:

- `needs_ocr: bool` — true ha `mime_type in {image/*}` VAGY `application/pdf + (size/page) heurisztika` (egyszerű: page_count becslés `size_bytes > 500_000 * page_count_hint`). Ha nincs page_count_hint, marad `needs_ocr=False` PDF-nél. Full scan-detekció S97+ scope, ne ess bele.
- `cloud_ai_allowed: bool` már megvan.

Új rule set (prepend, elé a mostani Rule 3-nak):

- **Rule 2.5 (Azure DI):** ha `needs_ocr == True` AND `cloud_ai_allowed == True` AND `AZURE_DOC_INTEL_ENDPOINT` env-ben elérhető → `chosen_parser="azure_document_intelligence"`, `fallback_chain=["docling_standard"]`, `reason="scan_pdf_cloud_allowed_azure_di"`.
- Különben marad a mostani logika.

Unit tesztek: új eset(ek) a `tests/unit/routing/test_multi_signal_router.py`-ban:

- scan PDF + cloud_ai_allowed=True + env set → azure_document_intelligence
- scan PDF + cloud_ai_allowed=False → docling_standard (nem skipped_policy, mert Docling tudja)
- scan PDF + cloud_ai_allowed=True + env MISSING → docling_standard (fallback)

### LÉPÉS 3 — `PIIRedactionGate` v0 (regex)

Fájl: `src/aiflow/guardrails/pii_redaction.py` (új)

- Class `PIIRedactionGate(name="pii_redaction_v0")`. Method `redact(text: str) -> PIIRedactionReport`.
- Regex v0: email (`\b[\w.+-]+@[\w-]+\.[\w.-]+\b`), magyar telefon (`\+?36[\s-]?\(?\d{1,2}\)?[\s-]?\d{3}[\s-]?\d{4}`), E.164 generic (`\+\d{8,15}`), IBAN (`\b[A-Z]{2}\d{2}[A-Z0-9]{10,30}\b`), TAJ (`\b\d{3}[\s-]?\d{3}[\s-]?\d{3}\b`).
- Output: `PIIRedactionReport(redacted_text: str, matches: list[{type, span, masked_value}], total_count: int)`. Pydantic v2.
- `PIIRedactionReport` schema egyelőre stub (Phase 2b / v1.5.1 hoz full provenance).
- **Nem** regisztráljuk még a `DocumentExtractorService.extract_from_package` LLM predig — az S97 (LLM field extraction) hozza. Itt csak a gate + teszt + DB stub.

Alembic 039 (zero-downtime): `pii_redaction_reports` tábla (`id uuid`, `extraction_id uuid nullable`, `match_count int`, `matches jsonb`, `created_at`). Indices: `ix_pii_redaction_extraction` on `extraction_id`. Downgrade: DROP.
